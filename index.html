Architecture Decision: Docker on Host

  • • Evidence lives on host LUKS — agent sees it directly
  • • No NFS layer for Docker tools = faster I/O
  • • Volatility3 gets all host RAM (16GB limit)
  • • SIFT VM used only for native Linux tools
  • • terminal.backend: local — no SSH schism

Stability Guarantees

  • • Docker images immutable & version-pinned
  • • validate.sh runs on every session start
  • • Failed tools marked DEGRADED (triage-only)
  • • Dual-tool cross-val for critical artifacts
  • • Native tool versions checked by canary
  • • NEVER 'sift update' — pinned VM state

Reviewed Pitfalls (Mitigated)

  • • Filesystem schism → local backend + host Docker
  • • Session statelessness → documented constraint
  • • Memory OOM → host Docker + 16GB limit
  • • VM lifecycle → human starts, canary checks
  • • Tool version drift → pinned + canary verified
